# 🤝 [2차 지시 - 안티 ➡️ 클코] PWA 완전 제거 및 리드와이즈 리더 테마 설계 요청서

날짜: 2026-05-27
버전: **v2.0 (피드백 반영)**

유저의 긴급 피드백을 반영하여 설계 방향을 전면 수정합니다.

현재 캐시 충돌 및 웹 성능 혼선을 일으키는 **PWA(Progressive Web App)를 아예 전면 제거**하고, 롤백 상태인 워크스페이스에 **'리드와이즈 리더(Readwise Reader)'** 테마를 가장 완벽하고 충돌 없는 방식으로 재설계해 주시기 바랍니다.

---

## 🛠️ 클코 2차 설계 핵심 요구사항

### 1. PWA 완전 제거 (PWA Strip & Unregister)
* **파일 삭제 명세**:
  - `manifest.json` [DELETE]
  - `sw.js` [DELETE]
* **HTML 소스 정리**:
  - `index.html`, `archive.html`, `about.html`, `search.html` 내부에 존재하는 `<link rel="manifest"...>`, `<meta name="theme-color"...>` 등 PWA 관련 메타 태그를 전부 제거합니다.
  - 기존 서비스 워커를 등록하는 자바스크립트 블록(`if ('serviceWorker' in navigator) { ... }`)을 완전히 삭제합니다.
* **클라이언트 강제 캐시 제거 스크립트 설계**:
  - 이미 기존 서비스 워커가 브라우저에 등록된 방문자들의 캐시 충돌을 막기 위해, 메인 페이지 로드 시 기존 등록된 서비스 워커를 **강제로 등록 해제(Unregister)**하는 안전 스크립트를 설계해 주세요:
    ```javascript
    if ('serviceWorker' in navigator) {
        navigator.serviceWorker.getRegistrations().then(function(registrations) {
            for(let registration of registrations) {
                registration.unregister();
            }
        });
    }
    ```

### 2. 스타일 격리 및 타이포그래피 재설계
* ** theme-modern.css 격리 설계**:
  - 기사 본문 페이지(`archive/*.html`)의 원래 본문 스타일이 깨지는 것을 막기 위해 `body.reader-mode` 하위 선택자 네임스페이스 격리 구조를 엄격하게 설계합니다.
  - 셸 페이지(`index.html`, `archive.html`, `about.html`)는 `body.shell-page` 네임스페이스를 사용하여 기존 레이아웃이 무너지지 않도록 조율합니다.
* **가독성 사양**: Coal Dark 테마 컬러(#121214), 680px 본문폭 정렬, Noto Sans KR + Outfit 가독성 폰트 매핑.

### 3. 고화질 헤더 이미지 자동 연동 파이프라인
* `scripts/post_process.py` 후처리기 설계: 기사 제목을 분석해 Unsplash Free API를 호출하는 로직 및 기사 내에 업로드된 이미지를 1순위로 자동 승격시키는 하이브리드 알고리즘 설계.

    - **최근 이미지 중복 차단 윈도우(Deduplication Window) 도입**:
      - 빌드 파이프라인 진행 시, **직전 3~5개 기사에서 사용된 이미지 ID를 메모리에 임시 기억하는 중복 차단 캐시 큐(Queue)**를 설계해 주세요.

      - 이를 통해 인접한 날짜나 동일 카테고리 기사 간에 **동일한 이미지 URL이 연달아 매칭되는 현상을 기술적으로 100% 완전 차단**하십시오.

## 🚀 4차 핵심 요구사항: 백엔드 고정형 하이브리드 뉴스 세그먼테이션 및 스마트 리딩 최적화
* **수정 대상**: `scripts/post_process.py` 내 뉴스 파서/매칭 시스템, `index.html` 내 최신 브리핑 렌더러 (`renderLatest`), GNB 테마 토글 추가 및 이미지 DB 풀
* **배경 및 전제조건**:
  - 원본 데이터의 구조(구글 시트/AI 생성 프로세스)는 고정되어 있어 백엔드 스키마는 절대 변경할 수 없습니다.
  - 현재 입력되는 데이터는 파이프 기호(`|`)로 결합된 `title`과 문장 단위의 줄글 `summary`로 구성되어 있습니다. 이를 역공학적으로 파싱하여 **글로벌 경제지 뉴스 포털 레이아웃**으로 재탄생시켜야 합니다.

* **작업 가이드**:

### 1. 하이브리드 뉴스 세그먼테이션 (Reverse-Engineering Parser)
* **타이틀 분할**: `title`의 파이프 기호(`|`)를 기준으로 문자열을 조각내어 깔끔한 개별 제목 배열을 만듭니다.
* **문맥 매칭 요약 청킹**:
  - 통짜 `summary`를 마침표(`. `) 기준으로 쪼개어 문장 배열을 확보합니다.
  - 각 타이틀 세그먼트에 포함된 핵심 고유 명사 키워드를 추출하여 문장과의 매칭 유사도(Noun Similarity)를 분석하거나, 차선책으로 발행 순서에 비례하여 개별 뉴스와 요약을 매칭합니다. 매칭되지 않은 잔여 문장은 자연스럽게 마지막 세그먼트에 병합해 정보 누락을 방지합니다.
* **언론사 1면(Front Page) 포털 그리드 레이아웃**:
  - `index.html` 내 `renderLatest` 함수를 개편하여, 1순위 핵심 뉴스는 큰 Hero 카드(이미지 배경 포함)로 구성하고, 2~3순위 뉴스는 2열 그리드(미니 썸네일 카드 형태)로 격자 배치합니다.
  - 모바일 해상도(Media Query)에서는 카드가 세로로 지나치게 길어지지 않도록 모바일에 적합한 컴팩트 리스트 레이아웃으로 반응형 스타일을 최적화해 주세요.

### 2. 시맨틱 이미지 미세 태그(Sub-tags) 매칭 및 메가 이미지 풀 확장
* **이미지 풀 메가 확장**: `CATEGORY_IMAGE_MAP` 내부의 각 테마별 Unsplash 고품질 이미지 개수를 기존 12개에서 **최소 24~30개 이상**으로 대폭 늘려(총 300장 규모) 다양성을 극대화합니다.
* **미세 태그 매칭 알고리즘**:
  - 이미지 메타데이터마다 세부 검색어 키워드 배열(`tags: ["rocket", "spacex", "fire", ...]`)을 부여합니다.
  - 1차로 상위 카테고리(예: `space`)를 감지한 후, 해당 풀 내의 이미지들을 돌며 기사 텍스트에 이미지 태그 키워드가 출현하는 빈도를 계산해 점수가 가장 높은 이미지를 **우선 순위 정밀 픽업(Smart Pick)**하여 기사에 매치합니다.
  - 이를 통해 로켓 관련 기사에는 화염을 뿜는 로켓 이미지가 뜨도록 맥락 오차를 완전히 차단합니다. 태그 스코어가 없거나 충돌할 때만 기존의 해시 모듈러 및 중복 방지 캐시 큐(`_dedup_window`)를 하위 안전장치(Fallback)로 가동합니다.

### 3. 메인 포털용 스마트 클래식 종이 모드 (라이트 모드) 하이브리드 토글
* **상세 기사 페이지**: 장시간 집중 정독을 위해 **다크 모드(Deep Reader Dark)를 영구 고수**합니다.
* **메인 및 아카이브 목록 페이지**:
  - 기본 접속 테마는 JFNB 고유의 **프리미엄 딥 다크 테마(Deep Nebula Dark)**를 유지하여 고급스러움을 유지합니다.
  - GNB(우상단) 메뉴바에 정갈한 아이콘 토글 버튼을 추가하여, 유저가 클릭 시 정통 인쇄 신문의 느낌을 살린 미색/화이트 바탕의 **클래식 종이 모드 (Warm-white Light Theme)**로 부드럽게 스위칭되도록 테마 전환 CSS 엔진을 완벽히 설계해 주세요.

* **피드백 내용 및 수정 지침**:

### 1) 상단 Hero 기사 영역 가로형 2분할 레이아웃 개편
* **현황**: 상단의 1순위 대표 Hero 기사 카드가 세로로 지나치게 크고, 은하수 배경 위에 글씨가 겹쳐 가독성이 떨어짐.
* **해결 지침**:
  - `.portal-hero` 클래스를 배경 오버레이형에서 **"좌측 이미지 50% + 우측 텍스트 50%" 가로형 분할 그리드(Grid)**로 전면 리디자인합니다.
  - `grid-template-columns: 1.2fr 1fr;` 또는 `flex` 방식을 사용하여, 좌측 절반에는 이미지가 찌그러짐 없이 칼같이 꽉 차게(`object-fit: cover;`) 흐르고, 우측 절반에는 정갈한 다크 카드 배경색(`var(--bg-card)`) 위에 기사 제목과 요약이 아름답게 나타나게 합니다.
  - 텍스트 위에 이미지가 겹치지 않으므로 글자의 시인성이 100% 보장됩니다.
  - 모바일 뷰(768px 이하)에서는 미디어 쿼리를 사용하여 가로 분할이 아닌 깔끔한 세로 배치 카드로 자연스럽게 스택 스위칭되도록 설계합니다.

### 2) 하단 서브 기사 카드 폰트 대폭 확대
* **현황**: 하단 서브 기사 2개(`.portal-sub`)의 제목 및 요약 텍스트 크기가 지나치게 작아 가독성이 떨어짐.
* **해결 지침**:
  - 서브 카드의 기사 제목 크기인 `.portal-sub-title`의 폰트 크기를 기존 `0.95rem` ➡️ **`1.15rem~1.2rem`** 내외로 대폭 확대 조정합니다.
  - 서브 카드의 요약글 크기인 `.portal-sub-summary`의 폰트 크기를 기존 `0.8rem` ➡️ **`0.9rem~0.95rem`** 수준으로 시원하게 키우고, 줄 간격(`line-height: 1.6`) 및 윗마진(`margin-top: 0.5rem`)을 주어 읽기 편리하게 개선합니다.

### 3) 셋 다 동일 우주 사진 노출 버그 정밀 진단 및 "세그먼트별 고유 이미지 빌드타임 주입" 아키텍처 수립
* **버그 원인**:
  - 현재 `index.html` 내 `renderLatest` 함수가 단 하나의 `briefing.thumb_url`만 가져와서 모든 서브 기사 영역에 `subSrc`로 일괄 적용하고 있음. 또한, `briefings.json` 빌드 시 최신 브리핑 데이터 객체에 각 세그먼트별 이미지를 쪼개서 담아주지 않고 통짜 1개만 정의했기 때문입니다.
* **해결 지침 (아키텍처 스펙)**:
  - **`post_process.py` 빌드 타임 파싱 확장**:
    - `post_process.py`가 데이터 파이프라인 후처리 시, 단순 텍스트 후처리를 넘어 **`briefings.json`에 저장할 데이터 스키마를 동적으로 확장**합니다.
    - 각 일일 브리핑 데이터 내부에 **`segments` 배열**을 신설하여, 빌드타임 단에서 타이틀 파이프(`|`) 파싱 및 요약 문장 분할을 선제적으로 수행하게 만듭니다.
    - 각 세그먼트 객체 내에 `title`, `summary`를 분할 저장하고, **세그먼트의 본문 텍스트에 맞춰 `detect_category` 및 시맨틱 미세 태그(Sub-tags) 이미지 매칭 시스템을 3회(각 세그먼트별로) 개별 가동**하여, 고유한 Unsplash 이미지 URL을 각 세그먼트의 `thumb_url` 필드로 정확히 꼽아줍니다.
    - *예시 산출 JSON 스펙*:
      ```json
      {
        "date": "2026-05-30",
        "title": "Blue Origin... | Dell... | Trump...",
        "segments": [
          { "title": "Blue Origin...", "summary": "...", "thumb_url": "우주사진URL" },
          { "title": "Dell AI 서버...", "summary": "...", "thumb_url": "반도체컴퓨터사진URL" },
          { "title": "Trump 협상안...", "summary": "...", "thumb_url": "금융유가사진URL" }
        ]
      }
      ```
  - **`index.html` 렌더러 개편**:
    - `renderLatest(briefing)` 함수에서는 복잡한 분할 파싱을 중복 수행하지 않고, JSON에 주입된 `briefing.segments` 배열을 그대로 매핑하여 렌더링합니다.
    - Hero 섹션은 `briefing.segments[0].thumb_url`을 사용하고, 서브 그리드의 각 카드들은 루프를 돌며 **`seg.thumb_url`을 각각 다르게 출력**하도록 바인딩합니다.
    - 이를 통해 3개의 서로 다른 뉴스가 각자의 시맨틱 키워드에 완벽하게 일치하는 다채로운 고유 이미지를 지니고 노출되도록 보장합니다.
